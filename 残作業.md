# 残作業

## 優先度A（リリース前に必ず完了する）
- [ ] `Installer/Package.wxs`：Manufacturer・Version・UpgradeCode のプレースホルダーを製品情報に置き換え、`COM_BLAS.dll`・`COMBLAS.tlb`・`libopenblas.dll`・ドキュメント類を payload に含める。WiX での COM 登録（`Class` 要素 or `Heat` 生成）を設定し、x64 専用のインストール先を `[ProgramFiles64Folder]` 配下に固定する。
- [ ] `COM_BLAS/COM_BLAS.rc`・`COM_BLAS/BLAS.rgs`・`COM_BLAS/COM_BLAS.rgs`：会社名・製品名・著作権・バージョンを正式値に更新し、ProgID を `Ckt.Com.Blas.BlasCore` 系へ揃える。TypeLib の version (`library` 1.3) と `FILEVERSION/PRODUCTVERSION` を一致させ、署名や公開年を反映させる。
- [ ] `COM_BLAS.sln`：`Release|x64` で `COM_BLASPS` プロジェクトに `Build.0` エントリを追加し、`msbuild COM_BLAS.sln /p:Configuration=Release /p:Platform=x64` が警告なしで proxy/stub をビルドできるようにする。
- [ ] `COM_BLAS/BLAS.cpp`：`ZIamax` 実装で発生している C4267（size_t→int 暗黙変換）警告を `CBLAS_INDEX` もしくは `auto` に置き換えて解消し、/WX 時でもビルドを通す。
- [ ] OpenBLAS 依存関係：`libopenblas.dll` をインストーラで配布し、ランタイム依存（`MSVCP140.dll` など VC++ 再頒布可能パッケージ）を前提条件として組み込むか同梱する。OpenBLAS の LICENSE を配布物に含める。
- [ ] リリース番号整合：MSI/ProductVersion、TypeLib version、RC の FileVersion/ProductVersion、ドキュメントに記載するバージョンを統一し、リリースノートのドラフトを用意する。

## 優先度B（リリースまでに仕上げたい改善）
- [ ] `Installer/Package.wxs` を単体ビルドできる CI/スクリプトを用意し（`devenv /build` もしくは WiX v4 の `wix` CLI）、`dotnet test` → `msbuild` → `wix build` の一連を自動化する。
- [ ] ドキュメント整備：`ReadMe.md` にサポート環境（64bit Windows、必要な再頒布パッケージ）、インストール後の COM 利用手順（VBA/.NET サンプル）、アンインストール手順を追記する。
- [ ] ライセンス/配布物：製品ライセンス文書と OpenBLAS の第三者ライセンスをルートまたはインストーラに同梱し、ReadMe からリンクする。
- [ ] コード署名フロー：`COM_BLAS.dll` と MSI を Authenticode 署名できるよう証明書取得手順と署名スクリプト（`signtool`）を整備する。
- [ ] テスト拡充：COM 経由の Z 系 API について、SAFEARRAY 下限が 1 以外のケースや Hermitian/Symmetric の in-place 更新後に `Complete*` 呼び出しが行われているかのリグレッションテストを追加する。
